# 大厅后续功能确认表

说明：`[ ]` 表示待做，`[x]` 表示已经确认或完成。后面开发时按这个表逐项勾。

## 账号与大厅

- [ ] 退出登录逻辑
  - 点击退出登录按钮。
  - 清空本地 token。
  - 跳转回登录页。
  - 防止返回大厅时继续显示旧用户信息。

- [ ] 点击切换头像逻辑
  - 点击头像后打开图片选择。
  - 先做本地预览。
  - 调接口上传头像并返回新的 `headImg` 地址。
  - 更新 token 或重新拉取用户信息，让大厅头像立即刷新。
  - 说明：不一定要上线才能做，本地也能开发；但真实保存头像必须有后端上传接口和图片存储位置。

## 开始游戏与房间

- [ ] 点击开始游戏后创建房间
  - 前端点击开始游戏。
  - 后端生成房间 `roomId`。
  - 前端跳转房间地址，例如 `/gameRoom/:roomId`。
  - 房间地址后面可以复制/分享给另一个玩家。

- [ ] 随机分配红方或黑方
  - 创建房间时随机给自己分配 `red` 或 `black`。
  - 另一个玩家进入后自动成为另一方。
  - 房间中只显示自己对应的视角。
  - 需要确认：是否允许观战。

- [ ] 加入 Socket
  - 进入房间后连接 socket。
  - 发送 `joinRoom`，参数包含 `roomId` 和用户信息。
  - 服务端维护房间状态、双方用户、当前棋盘、当前回合。
  - 走棋后通过 socket 广播给双方。
  - 断线重连后能恢复房间状态。
  - 这块需要重点配合拆逻辑。

## 对局记录与回放

- [ ] 查看战绩
  - 展示自己的历史对局。
  - 展示胜负、红黑方、对手、时间、总步数。
  - 支持点击进入某一局详情。

- [ ] 记录每一步 JSON
  - 每一步保存为 JSON。
  - 记录移动前位置、移动后位置、棋子、吃子信息、当前方、时间。
  - 对局结束时保存完整棋谱。

- [ ] 自动播放回放
  - 打开对局详情后可以自动播放。
  - 支持暂停、继续、重新播放。
  - 支持调整播放速度。

- [ ] 上一步 / 下一步
  - 点击上一手回退棋盘。
  - 点击下一手前进棋盘。
  - 当前步数要有明显展示。

## 排行榜

- [ ] 胜率排行榜
  - 展示所有玩家排行。
  - 按胜率排序。
  - 同胜率时可按胜场数排序。
  - 展示昵称、头像、胜率、胜场、负场、总场次。
  - 需要过滤总场次太少的玩家，避免 1 胜 0 负直接第一。
